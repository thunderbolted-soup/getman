# Getman: Context для AI-агента (Pro Version)

## Обзор проекта
Getman — профессиональный десктопный HTTP-клиент, ориентированный на локальную работу. Основная цель: быстрая отладка API без облачной синхронизации. Поддерживает многовкладочность, импорт из различных форматов (Postman, cURL) и расширение через Python-скрипты.

## Техстек
- **Core:** Python 3.11+, PySide6 (UI)
- **HTTP Engine:** `httpx` (асинхронные запросы через `HttpClientThread`). Поддержка multipart/form-data.
- **Scripting:** Python `exec()` для Pre-request скриптов.
- **Storage:** Локальные JSON-файлы в `data/`.

## Архитектура
- `main.py` — точка входа.
- `core/`:
    - `request_preparer.py` [CRITICAL]: Центральный узел подготовки запроса. Обрабатывает `apply_env` (рекурсивно), выполняет скрипты, накладывает Auth и генерирует cURL.
    - `http_client.py`: Асинхронное выполнение запросов с поддержкой отмены (cancel).
    - `environment.py`: Управление переменными окружения.
- `ui/`:
    - `main_window.py`: Управление вкладками, глобальный поиск (debounced), настройки.
    - `request_panel.py`: Редактор запроса с вкладками Params/Headers/Body/Auth/Pre-request.
    - `shared_widgets.py`: Универсальный `BaseTable` для всех табличных данных.
    - `json_editor.py`: `CodeEditor` (JSON) и `PythonCodeEditor` (Scripts) с подсветкой и валидацией.

## Правила кодинга
- **Подготовка запроса:** Никогда не дублировать логику подготовки в UI. Всегда использовать `prepare_request` из `core.request_preparer`.
- **Асинхронность:** Все сетевые запросы ТОЛЬКО через `HttpClientThread`. Не блокировать UI поток.
- **Переменные:** Поддержка `{{variable}}` обязательна для всех полей запроса через рекурсивный `apply_env`.
- **UI:** 
    - Использовать `QSplitter` для адаптивности. 
    - Поиск по коллекциям/истории должен быть debounced (через `QTimer`).
    - Поддержка горячих клавиш (Ctrl+T, Ctrl+W, Ctrl+S, Ctrl+Return) обязательна.
    - Использовать динамические свойства Qt (`setProperty`) для индикации ошибок (например, `error="true"`).

## Важно
- **Никаких облаков:** Вся работа должна быть строго локальной.
- **Экспорт:** Поддержка генерации cURL из текущего запроса должна учитывать переменные окружения и изменения из Pre-request скриптов.
- **UX:** Кнопка "Send" должна превращаться в "Cancel" во время выполнения запроса.
- **Response:** JSON-ответы должны форматироваться автоматически. Подсветка синтаксиса — Monokai.
- **Network:** Сетевые ошибки должны быть человекочитаемыми.
