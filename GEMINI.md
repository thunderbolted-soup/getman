# Getman: Context для AI-агента

## Обзор проекта
Getman — это минималистичный десктопный HTTP-клиент, ориентированный на локальную работу. Основная цель: быстрая отладка API без облачной синхронизации.

## Техстек
- **Core:** Python 3.11+, PySide6 (UI)
- **HTTP:** `httpx` (асинхронные запросы через `QThread`)
- **UI Utils:** `pygments` (подсветка JSON/XML)
- **Storage:** Локальные JSON-файлы в `data/`

## Архитектура
- `main.py` — точка входа и инициализация приложения.
- `core/` — бизнес-логика:
    - `http_client.py` — асинхронное выполнение запросов.
    - `auth.py` — управление профилями авторизации.
    - `collection.py` — импорт и обработка коллекций.
    - `logger.py` — система логирования с сигналами для UI.
- `storage/` — работа с данными:
    - `store.py` — низкоуровневый JSON I/O.
    - `history.py` — лог истории запросов.
- `ui/` — компоненты интерфейса (панели запроса, ответа, истории и логов).

## Правила кодинга
- **Асинхронность:** Все сетевые запросы ТОЛЬКО через `HttpClientThread`. Не блокировать UI поток.
- **Логирование:** Использовать `core.logger.get_logger()`. Никаких `print()`.
- **UI:** Использовать `QSplitter` для адаптивности. Таблицы для Key-Value параметров.
- **Данные:** Всегда использовать обертки из `storage.store` для записи файлов.
- **Стиль:** PEP 8, обязательные type hints для новых функций.

## Команды
```bash
python main.py        # запуск приложения
pip install -r reqs.txt # установка зависимостей
cat data/app.log      # просмотр сырых логов
```

## Важно
- **Никаких облаков:** Вся работа должна быть строго локальной.
- **Совместимость:** Поддержка импорта Postman Collection v2.1 обязательна.
- **Логи:** Каждое действие пользователя (Send, Import, Delete) должно логироваться.
- **Очистка:** Удаление коллекции или истории требует логирования уровня `WARNING`.
