# Getman: Context для AI-агента

## Обзор проекта
Getman — это минималистичный десктопный HTTP-клиент, ориентированный на локальную работу. Основная цель: быстрая отладка API без облачной синхронизации. Поддерживает многовкладочность и импорт из различных форматов (Postman, cURL).

## Техстек
- **Core:** Python 3.11+, PySide6 (UI)
- **HTTP:** `httpx` (асинхронные запросы через `QThread`)
- **UI Utils:** `pygments` (подсветка JSON/XML/HTML)
- **Storage:** Локальные JSON-файлы в `data/`

## Архитектура
- `main.py` — точка входа и инициализация приложения.
- `core/` — бизнес-логика:
    - `http_client.py` — асинхронное выполнение запросов.
    - `auth.py` — управление профилями авторизации.
    - `collection.py` — импорт и обработка коллекций.
    - `logger.py` — система логирования с сигналами для UI.
- `storage/` — работа с данными:
    - `store.py` — низкоуровневый JSON I/O.
    - `history.py` — лог истории запросов.
- `ui/` — компоненты интерфейса:
    - `main_window.py` — управление вкладками, импорт cURL, общее окно.
    - `request_panel.py` — редактор запроса (Params/Headers/Body/Auth).
    - `response_panel.py` — просмотрщик ответов с авто-форматированием JSON.
    - `log_viewer.py` — real-time монитор системных логов.

## Правила кодинга
- **Асинхронность:** Все сетевые запросы ТОЛЬКО через `HttpClientThread`. Не блокировать UI поток.
- **Логирование:** Использовать `core.logger.get_logger()`. Никаких `print()`.
- **UI:** 
    - Использовать `QSplitter` для адаптивности. 
    - Кнопки добавления вкладок располагать в `cornerWidget` объекта `QTabWidget`.
    - Сплиттеры должны иметь визуальный hover-эффект (стилизация через QSS).
    - Поддержка горячих клавиш (Ctrl+T, Ctrl+W, Ctrl+S, Ctrl+Return) обязательна.
    - Двойной клик по вкладке должен позволять её переименовать.
- **Данные:** Всегда использовать обертки из `storage.store` для записи файлов.
- **Стиль:** PEP 8, обязательные type hints для новых функций.

## Команды
```bash
python main.py             # запуск приложения
pip install -r requirements.txt # установка зависимостей
cat data/app.log           # просмотр сырых логов
pyinstaller Getman.spec    # сборка Portable EXE (на Windows)
```

## Важно
- **Никаких облаков:** Вся работа должна быть строго локальной.
- **Совместимость:** Импорт Postman Collection v2.1 и cURL команд обязателен.
- **Экспорт:** Поддержка генерации cURL из текущего запроса (Copy as cURL) обязательна.
- **История:** Клик по истории открывает запрос в новой вкладке.
- **Логи:** Каждое значимое действие (Send, Import, Save, Delete) должно логироваться.
- **Response:** 
    - JSON-ответы должны форматироваться (prettify) автоматически при получении.
    - Кнопка "Wrap" должна работать во всех режимах просмотра (Pretty/Raw).
    - Подсветка синтаксиса в режиме Pretty должна использовать тему Monokai.
- **Network:** Сетевые ошибки должны быть человекочитаемыми (например, "Could not resolve hostname").
- **Agent:** Исходящие запросы должны иметь дефолтный `User-Agent: Postman/7.0.0`.
- **UX:** При закрытии приложения все дочерние окна (Log Viewer) должны закрываться.
